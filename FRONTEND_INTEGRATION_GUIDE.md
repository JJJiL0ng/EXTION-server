# 프론트엔드 통합 가이드

## 현재 문제점

프론트엔드에서 보내는 데이터 구조에 문제가 있어 수정이 필요합니다.

### 1. 헤더 데이터 문제

**현재 (잘못된 구조):**
```json
{
  "name": "뒤풀이명단",
  "headers": [
    "202531194",
    "전채은", 
    "경영학과",
    "01028791015",
    "O"
  ],
  "data": [
    [
      "202531299",
      "김현지",
      "관광경영학과",
      "010-4695-6237",
      "O"
    ]
  ]
}
```

**수정해야 할 구조:**
```json
{
  "name": "뒤풀이명단",
  "headers": [
    "학번",
    "이름",
    "학과", 
    "전화번호",
    "참석여부"

  "data": [

      "경영학과", 
      "01028791015",
      "O"

    [
      "202531299",
      "김현지",
      "관광경영학과",
      "010-4695-6237",
      "O"
    ]
  ]
}
```

### 2. 데이터 구조 설명

- **headers**: 실제 컬럼명 (학번, 이름, 학과, 전화번호, 참석여부)
- **data**: 첫 번째 행부터 모든 실제 데이터 포함
- **첫 번째 행도 데이터에 포함**: 현재 headers에 들어있는 첫 번째 행 데이터도 data 배열에 포함되어야 함

## 수정 작업

### 1. 헤더 추출
- 스프레드시트의 첫 번째 행에서 실제 컬럼명을 추출
- 예: ["학번", "이름", "학과", "전화번호", "참석여부"]

### 2. 데이터 정리
- 현재 headers에 들어있는 데이터를 data 배열의 첫 번째 요소로 이동
- 모든 실제 데이터 행을 data 배열에 포함

### 3. 검증
- headers 배열의 길이와 각 data 행의 길이가 일치하는지 확인
- 모든 데이터가 문자열 형태인지 확인

## Firebase 권한 문제

현재 Firebase 저장 시 "스프레드시트 접근 권한이 없습니다" 오류가 발생하고 있습니다. 이는 다음과 같은 원인일 수 있습니다:

1. **잘못된 spreadsheetId**: chatId와 실제 스프레드시트 ID가 다를 수 있음
2. **권한 설정**: Firebase 규칙에서 해당 사용자의 스프레드시트 접근 권한 확인 필요
3. **데이터 구조**: 잘못된 데이터 구조로 인한 저장 실패

## 테스트 방법

1. 수정된 데이터 구조로 API 호출
2. 백엔드 로그에서 다음 확인:
   - 올바른 헤더가 파싱되는지
   - 데이터 행 수가 정확한지
   - Firebase 저장이 성공하는지
